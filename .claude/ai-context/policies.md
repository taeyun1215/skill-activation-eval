# 주요 정책 (Main Policies)

## 1. 신고 정책 (Report Policy)

### 신고 유형
| 유형 | 설명 |
|------|------|
| SPAM | 스팸/광고 |
| INAPPROPRIATE | 부적절한 콘텐츠 |
| HARASSMENT | 괴롭힘 |
| FAKE_PROFILE | 허위 프로필 |
| SCAM | 사기 |

### 신고 처리 로직
- 동일 사용자 재신고: **7일 후** 가능
- 신고 누적 **3회** → 경고 (WARNING)
- 신고 누적 **5회** → 임시 차단 (TEMPORARY)
- 심각한 위반 → 영구 차단 (PERMANENT)

### 차단 해제
- 경고/임시차단: **6개월 후** 자동 해제
- 영구차단: 관리자 수동 검토 필요

### 신고 대상
- 사용자 프로필
- 모먼트/포스트
- 댓글/대댓글
- 채팅 메시지

## 2. 회원 정책 (Member Policy)

### 가입 조건
- SNS 인증 필수 (Apple, Google, Kakao, Line, Naver)
- 만 18세 이상

### 본인 인증
| 방식 | 설명 | 적용 국가 |
|------|------|-----------|
| PASS 인증 | 휴대폰 본인인증 | 한국 |
| 신분증 인증 | Argos OCR + 얼굴 매칭 | 전체 |

### 프로필 필수 항목
- 닉네임 (자동 생성 또는 직접 입력)
- 생년월일
- 성별
- 프로필 이미지 (최소 1장)

### 탈퇴 정책
- 즉시 탈퇴 가능
- 데이터 보관: 30일 후 완전 삭제
- 재가입: 탈퇴 후 7일 이후 가능

## 3. 통화 정책 (Call Policy)

### 통화 플로우
```
매칭 요청 → 대기열 진입 → 매칭 성공 → Agora RTC 연결 → 통화 시작
                              ↓
                         매칭 실패 (타임아웃)
```

### 랜덤 통화 규칙
- 최대 대기 시간: 60초
- 동일 상대 재매칭 방지: 24시간
- 차단/신고 상대 제외
- 통화 중 재매칭 불가

### 통화 종료 후
- 상호 평가 (선택)
- 채팅방 자동 생성 (7분 이상 통화 시)
- 리워드 지급 조건 충족 시 풍선 지급

### 통화 평가 항목
- GOOD_VOICE (좋은 목소리)
- KIND (친절함)
- FUN (재미있음)
- GOOD_LISTENER (좋은 청자)
- MORE_TALK (더 대화하고 싶음)

## 4. 매칭 정책 (Matching Policy)

### 매칭 알고리즘
```kotlin
매칭 점수 = 언어호환도 * 0.3 + 활성도 * 0.2 + 프로필완성도 * 0.2 + 거리 * 0.15 + 선호도일치 * 0.15
```

### 매칭 조건
| 조건 | 필터링 |
|------|--------|
| 성별 | 사용자 설정 (전체/남성/여성) |
| 나이 | ±10세 기본, 커스텀 가능 |
| 언어 | 공통 언어 1개 이상 |
| 차단 | 상호 차단 시 제외 |
| 신고 | 신고 이력 시 제외 |

### 나이 매칭 규칙
- 미성년자(18-19세): 동일 연령대만 매칭
- 성인: 설정 범위 내 매칭

### 매칭 모드
| 모드 | 설명 |
|------|------|
| RANDOM | 완전 랜덤 매칭 |
| NEARBY | 근처 사용자 우선 |
| LANGUAGE | 동일 언어 우선 |

## 5. 채팅 정책 (Chat Policy)

### 여성 일일 랜덤채팅 리밋
| 조건 | 일일 제한 |
|------|-----------|
| 풍선 구매 60일 이내 | 무제한 |
| 구매 이력 없음 (60일 초과) | 하루 5회 |

- Redis 키: `daily-random-chat-limiter:range` (ZSet, score = 사용 횟수)
- 광고 보상(adreward)으로 리밋 해제 가능
- GraphQL: `hasDailyRandomChatLimits` → `Boolean!`

### 랜덤 채팅
- 통화 후 자동 생성 (7분 이상 통화 조건)
- 1:1 채팅만 지원

### 일반 채팅
- 팔로우 관계 없이 가능
- 메시지 저장: DynamoDB (TTL 1년)
- 메타데이터: MySQL

### 메시지 기능
- 텍스트 (최대 1000자)
- 이미지 (최대 10장)
- 음성 메시지 (최대 60초)
- 번역 (AWS Translate)

### 금지어 필터
- 실시간 필터링
- 언어별 금지어 목록 관리
- 위반 시 메시지 발송 차단

### 인앱 결제

|--------|-----------|

| 획득 방법 | 유효기간 |
|-----------|----------|

| 일일 미션 | **14일** |
| 광고 시청 | **14일** |
| 이벤트 지급 | 이벤트별 상이 |

### 일일 미션 풍선
| 미션 | 보상 |
|------|------|
| 미션 1 (앱 접속) | 5개 |
| 미션 2 (프로필 확인) | 10개 |
| 미션 3 (모먼트 작성) | 10개 |
| 미션 4 (통화 1회) | 15개 |
| 미션 5 (통화 3회) | 15개 |
| 미션 6 (7분 통화) | 20개 |
| 미션 7 (리뷰 작성) | 30개 |

### 티켓 시스템
- 통화 시작 시 티켓 1개 소모
- 구독자: 무제한 티켓
- 비구독자: 풍선으로 티켓 구매

### 환불 정책
- 미사용 풍선: 전액 환불 가능
- 부분 사용: 사용분 제외 환불
- 구독: 플랫폼 정책 따름

### 구독 상태
```
ACTIVE → GRACE_PERIOD (3일) → EXPIRED
    ↑__________|  (결제 성공 시 복구)
```

## 7. 푸시알림 정책 (Push Notification Policy)

### 푸시 유형
| 유형 | 우선순위 | 설명 |
|------|----------|------|
| CHAT | 높음 | 새 메시지 |
| CALL_MATCH | 최고 | 통화 매칭 |
| FOLLOW | 중간 | 새 팔로워 |
| LIKE | 낮음 | 좋아요 |
| COMMENT | 중간 | 댓글 |
| SERVICE | 최고 | 시스템 알림 |

### 활동 알림 (Activity)
- 팔로우 알림
- 모먼트/포스트 좋아요
- 댓글/대댓글
- 프로필 방문

### 알림 설정
| 설정 | 기본값 |
|------|--------|
| 전체 알림 | ON |
| 채팅 알림 | ON |
| 팔로우 알림 | ON |
| 좋아요 알림 | ON |
| 마케팅 알림 | OFF |

### 무음 시간
- 사용자 설정 가능 (예: 22:00 ~ 08:00)
- 긴급 알림(CALL_MATCH)은 무음 시간에도 발송

### 최적 알림 시간
- 사용자 활성 시간 분석
- 지역 시간대 고려
- 개인화된 발송 시간 적용

## 8. 스캠 대응 정책 (Scam Prevention)

### 스캠 감지 기준
| 유형 | 감지 조건 |
|------|-----------|
| IP 기반 | 동일 공개 IP에서 다수 계정 |
| 프로필 기반 | 신분증 미인증 + 의심 패턴 |
| 통화 기반 | 짧은 시간 다수 통화 종료 |
| 행동 기반 | 비정상적 메시지 패턴 |

### 대응 조치
1. 자동 모니터링 목록 추가
2. 신분증 인증 요청
3. 임시 기능 제한
4. 심각한 경우 계정 차단

### 신분증 확인 요청 조건
- 스캠 의심 점수 임계값 초과
- 다수 사용자로부터 신고
- 관리자 수동 요청
