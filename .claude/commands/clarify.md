# Clarify — 소크라테스식 질문

## 목적

모호한 요구사항을 명확하게 만들기 위해 질문을 던진다.
코드를 작성하기 전에, 무엇을 만들어야 하는지 정확히 이해한다.

## 절차

1. **$ARGUMENTS** 분석
2. 관련 코드베이스 탐색 (현재 구현 상태 파악)
3. 다음 관점에서 질문 생성:

### 질문 카테고리

- **범위**: "이 기능의 경계는 어디인가? 어떤 모듈에 영향을 주는가?"
- **사용자**: "이 기능의 사용자는 누구인가? 어떤 상황에서 사용하는가?"
- **엣지 케이스**: "입력이 없거나, 잘못된 경우는? 동시 요청은?"
- **의존성**: "이 변경이 다른 서비스에 영향을 주는가? Proto 변경이 필요한가?"
- **기존 패턴**: "이미 유사한 기능이 있는가? 어떤 패턴을 따르는가?"
- **테스트**: "이 기능을 어떻게 검증할 수 있는가?"

## 출력 형식

```
## 현재 이해

[요구사항에 대한 현재 이해 정리]

## 확인 질문

1. [질문 1]
2. [질문 2]
3. ...

## 관련 코드

- `파일경로:라인` — 설명
```

## 원칙

- 답을 가정하지 않는다
- 코드를 먼저 작성하지 않는다
- 질문은 구체적이고 실행 가능해야 한다
- 최대 5개 질문으로 제한한다
